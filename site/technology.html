Core Principles

Engineering standards behind the 2.0 line

Zero-GC Rendering Discipline

  • Renderer paths optimized for low transient allocation
  • Static-frame reuse strategies for heavy fill workloads
  • Stable frame pacing in sustained sessions

Thread-Safe Model Access

  • Background writers with EDT-consistent reads
  • Compatibility checks for renderer/model pairing
  • Defensive fallbacks instead of undefined states

HiDPI and Visual Fidelity

  • Crisp rendering strategy for retina and 4K targets
  • Calibration-sensitive domains kept scale-stable
  • Consistent color and theme behavior across renderers
Architecture Snapshot

How teams typically integrate ArberCharts 2.0

Data and Model Layer

Domain streams feed specialized models. Models maintain update stamps and are prepared for concurrent access patterns. 

model.append(sample);
if (model.getUpdateStamp() != lastStamp) {
  panel.repaint();
}

Rendering and UI Layer

Renderers are attached through ArberChartBuilder and controlled with theme- and compatibility-aware defaults for predictable output. 

ArberChartPanel panel = ArberChartBuilder.of()
    .addLayer(model, renderer)
    .withTheme(ChartThemes.getDarkTheme())
    .buildPanel();
