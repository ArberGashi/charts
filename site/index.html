Technical Excellence

Zero-GC Konzept

Die Herausforderung: Determinismus in Java

In Echtzeitsystemen ist der Stop-the-World-GC der grösste Feind. Wenn im Render-Loop ständig neue Objekte entstehen, sind Pausen unvermeidlich.

Der ArberCharts-Ansatz

  • Allocation-Free Rendering: keine neuen Objekte im aktiven Zeichnen.
  • Primitive-basierte Verarbeitung statt Boxing/Unboxing.
  • Object Pooling für wiederverwendbare Ressourcen.
  • Java 25 Native Performance für stabile Frame-Rates.

Ergebnis: glattes Schriftbild und flüssige Animationen ohne Mikroruckler – auch nach stundenlangem Dauerbetrieb.
