# v1.7.0 Breaking Changes

This release removes legacy compatibility shims and enforces the High-Integrity Architecture
Doctrine across all layers.

## Summary

- `com.arbergashi.charts.ui.*` is removed entirely.
- **v1.5.0 Clean Core Edition note:** deprecated `com.arbergashi.charts.ui.ArberChartPanel` and
  `com.arbergashi.charts.ui.ChartExportService` were removed early as part of the Architecture
  Doctrine cleanup. Use the platform equivalents immediately.
- Domain is strictly headless (no AWT/Swing types).
- Doctrine enforcement is fail-fast in the build (ArchUnit violations break the build).

## Package Removals and Replacements

| Removed (v1.7.0) | Replacement |
| --- | --- |
| `com.arbergashi.charts.ui.ArberChartPanel` | `com.arbergashi.charts.platform.swing.ArberChartPanel` |
| `com.arbergashi.charts.ui.ChartExportService` | `com.arbergashi.charts.platform.export.ChartExportService` |
| `com.arbergashi.charts.ui.legend.*` | `com.arbergashi.charts.render.legend.*` |
| `com.arbergashi.charts.ui.grid.*` | `com.arbergashi.charts.render.grid.*` |
| `com.arbergashi.charts.medical.*` | `com.arbergashi.charts.domain.medical.*` |
| `com.arbergashi.charts.predictive.*` | `com.arbergashi.charts.engine.predictive.*` |
| `com.arbergashi.charts.forensic.*` | `com.arbergashi.charts.engine.forensic.*` |

## Behavior Changes

- **No UI types in domain**: AWT/Swing types are removed from domain objects. Replace any
  `Color`, `Rectangle`, or `Insets` references with ARGB/int or domain primitives.
- **Strict layer enforcement**: ArchUnit rules are now fail-fast. Builds will fail if
  layer boundaries are violated.

## Migration Prerequisites

- Update imports per `MIGRATION_GUIDE_v1.7.md`.
- Ensure custom renderers use `platform.render.StrokeCache` and `platform.render.ColorCache`.

## Timeline

- v1.5.x: Compatibility bridge available (deprecated).
- v1.7.0 LTS: Compatibility bridge removed.
